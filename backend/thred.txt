## Общая цель проекта

HiveMonitor - система мониторинга пасек и ульев:

- firmware ESP8266/ESP32 публикует telemetry/events/status через MQTT;
- backend на Go принимает MQTT, хранит PostgreSQL, отдает REST API;
- web frontend на React/Vite показывает организации, пасеки, ульи, устройства, телеметрию, события;
- mobile будет позже;
- проект должен расти в SaaS с тарифами, ролями, уведомлениями, AI, weather analytics, OTA.

## Текущее состояние backend/firmware/web

Backend:

- Go + chi + pgx + goose.
- Docker Compose: Postgres, NATS, Mosquitto, api-service, mqtt-ingestion-service.
- Реализованы auth, organizations, apiaries, hives, unassigned devices, assign device, telemetry latest/history, apiary/hive events.
- `GET /hives/{hiveID}/telemetry/history` уже принимает `from/to/limit`.
- mqtt-ingestion принимает telemetry/events/status.

Firmware:

- есть apiary-aware topics:
  - `apiaries/{apiaryId}/devices/{deviceId}/telemetry`
  - `apiaries/{apiaryId}/devices/{deviceId}/events`
  - `apiaries/{apiaryId}/devices/{deviceId}/status`
- legacy topics `hives/{deviceId}/...` оставлены как compatibility mode.

Web:

- React + TypeScript + Vite.
- До рефакторинга почти все было в `web/src/App.tsx`.
- Сейчас `App.tsx` стал контейнером данных/orchestration, большая часть UI вынесена в:
  - `web/src/components/DashboardParts.tsx`
  - `web/src/components/MapProvider.tsx`
- Добавлены static assets в `web/public/images/...`.
- Старые root-level assets удалены:
  - `web/public/images/apiary-meadow.svg`
  - `web/public/images/hive-stack.svg`
- Добавлен `leaflet` и `@types/leaflet`.

### 7. Новый экран `Пасеки`

Пользователь и Codex провели Q&A и приняли решения:

- экран `Пасеки` - отдельный пункт навигации;
- MVP без React Router, переключение view через state sidebar;
- представитель организации видит только пасеки выбранной организации;
- по умолчанию ничего не выбрано, пока пользователь не кликнет на пасеку;
- список пасек показывает:
  - название;
  - регион/населенный пункт;
  - основной статус;
  - отдельный бейдж `Нужна привязка`, если есть unassigned devices;
  - количество ульев;
  - активные ульи;
  - алерты/события;
  - `последнее обновление` НЕ показывать;
- карта должна быть реальная;
- заложить abstraction `map_provider`;
- первый provider: OpenStreetMap/Leaflet;
- правая панель выбранной пасеки показывает только сводку и кнопку `Перейти к пасеке`;
- в правой панели адрес/регион крупно, координаты мелко вторичной строкой;
- кнопка `Перейти к пасеке` выбирает пасеку и переключает на текущий dashboard `Обзор пасеки`;
- на экране есть кнопка `Новая пасека`;
- она открывает полноценную форму создания пасеки;
- поля формы:
  - название;
  - описание;
  - страна;
  - область/регион;
  - населенный пункт;
  - адрес;
  - описание места;
  - широта;
  - долгота;
  - часовой пояс;
- форма позволяет выбрать точку на карте кликом;
- клик по карте заполняет latitude/longitude;
- при клике пробовать reverse geocoding через OpenStreetMap/Nominatim;
- если reverse geocoding не сработал, оставить адресные поля пустыми для ручного ввода;
- большая карта и mini-map в форме используют одну provider abstraction.
