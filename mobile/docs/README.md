# Mobile

## Назначение

Мобильные приложения Android и iOS дают быстрый доступ к состоянию пасек и ульев, задачам, событиям и уведомлениям.

## Стек

- React Native;
- TypeScript.

## Языки

Мобильные приложения должны поддерживать:

- русский;
- украинский;
- английский.

## MVP под текущий backend

Мобильное приложение может использовать те же REST endpoints, что и web:

- регистрация и вход;
- список организаций;
- список пасек;
- список ульев;
- карточка улья;
- latest telemetry;
- history telemetry;
- список непривязанных устройств;
- привязка устройства к улью.

Push-уведомления пока не реализованы на backend-стороне. До появления `notification-service` мобильное приложение может показывать текущее состояние и историю.

## Provisioning устройств

Текущий MVP:

- пользователь настраивает устройство через локальную AP и web-интерфейс firmware;
- в mobile можно показывать MQTT credentials пасеки, когда backend добавит API для credentials;
- привязка устройства выполняется после появления устройства в backend как `unassigned`.

Будущий сценарий:

- поиск нового устройства приложением;
- передача Wi-Fi и MQTT настроек устройству;
- QR/BLE/Wi-Fi provisioning;
- автоматическая привязка к пасеке.

## Уведомления

Целевые каналы по ТЗ:

- push;
- Telegram;
- in-app.

Для спящих устройств alert "offline" должен быть заменен логикой "пропущено N плановых передач".

## После текущего MVP

- Tasks/reminders.
- Журнал событий.
- Настройки уведомлений.
- Схема ульев на пасеке.
- Weather analytics.
- AI insights.
- OTA управление, если будет разрешено ролью.
