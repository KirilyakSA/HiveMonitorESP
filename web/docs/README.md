# Web

## Назначение

Web-интерфейс системы используется для управления пасеками, ульями, устройствами, просмотра телеметрии, событий и уведомлений.

## Стек

- React;
- TypeScript;
- Vite.

Web-интерфейс реализуется как SPA.

## Языки

Web-интерфейс должен поддерживать:

- русский;
- украинский;
- английский.

## Основные экраны

- Вход в систему.
- Дашборд пасеки.
- Список пасек.
- Список ульев.
- Карточка улья.
- Графики измерений.
- Журнал событий.
- Настройки устройства.
- Настройки уведомлений.

## Карточка улья

Карточка улья должна показывать:

- текущий вес;
- изменение веса за выбранный период;
- температуру;
- влажность;
- статус открытия;
- батарею: процент и напряжение;
- состояние связи;
- время последнего измерения;
- последние события.

## MVP

Для MVP web-интерфейс должен поддерживать:

- вход пользователя;
- роль владельца пасеки;
- роль администратора системы;
- список ульев;
- карточку одного улья;
- графики веса, температуры и влажности;
- просмотр базовых событий;
- ручную привязку устройства к улью по `deviceId`;
- генерацию `deviceToken`;
- базовые настройки устройства без удаленного изменения пинов.

## После MVP

- Помощники и шаринг доступа.
- Расширенная настройка устройств.
- Удаленное изменение пинов.
- QR-привязка устройств.
- Разрешение конфликтов конфигурации через UI.
- Управление OTA через backend.
