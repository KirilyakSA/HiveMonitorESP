# Web

## Назначение

Web-интерфейс системы используется для управления организациями, пасеками, ульями, устройствами, просмотра телеметрии, задач, событий и уведомлений.

## Стек

- React;
- TypeScript;
- Vite.

Web-интерфейс реализуется как SPA.

## Языки

Web-интерфейс должен поддерживать:

- русский;
- украинский;
- английский.

## MVP-экраны под текущий backend

- Регистрация и вход.
- Список организаций.
- Создание организации.
- Список пасек.
- Создание пасеки.
- Список ульев пасеки.
- Создание улья.
- Список непривязанных устройств пасеки.
- Привязка устройства к улью.
- Выбор действия со старой телеметрией устройства:
  - привязать к улью;
  - удалить;
  - оставить диагностической.
- Карточка улья.
- Latest telemetry.
- History telemetry и базовые графики.

## Карточка улья

Карточка улья должна показывать:

- текущий вес;
- изменение веса за выбранный период;
- температуру;
- влажность;
- статус открытия;
- батарею: процент и напряжение;
- время последнего измерения;
- состояние устройства по пропущенным плановым передачам, а не по простому offline.

## Provisioning устройств

Актуальная модель:

- MQTT credentials выдаются на уровне пасеки;
- пользователь сбрасывает устройство;
- устройство поднимает AP;
- пользователь вводит Wi-Fi и MQTT credentials через локальный web-интерфейс firmware;
- устройство появляется в backend как `unassigned`;
- админ пасеки привязывает устройство к улью.

Текущая firmware публикует legacy topic `hives/{deviceId}/telemetry`. Для автоматического provisioning по пасеке нужен следующий firmware-инкремент с `apiaryId` или topic prefix.

- Настройки alert rules на уровне пасеки/улья.
- Настройки уведомлений push/Telegram.
