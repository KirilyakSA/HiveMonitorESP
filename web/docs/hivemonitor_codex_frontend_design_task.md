# Задание для Codex / разработчика: редизайн frontend HiveMonitor

## Цель

Переделать текущий `web` frontend HiveMonitor в современную рабочую панель пасеки.

Главный экран должен быть не лендингом, а полноценным dashboard для пасечника/администратора пасеки:

- сводка по пасеке;
- предупреждения/алерты;
- события;
- список ульев с ключевой информацией;
- непривязанные устройства;
- текущая погода и погодные предупреждения;
- советы/подсказки;
- сравнительный график изменения веса всех ульев;
- отдельная боковая панель выбранного улья.

Дизайн должен быть похож на современный SaaS-dashboard с мягкой пасечной темой: светлый интерфейс, honey/amber акценты, спокойный фон, карточки, таблицы, графики, понятные статусы.

# 2. Технические ограничения

Важно:

1. Не менять backend API.
2. Не менять текущие endpoint-адреса.
3. Не ломать текущую auth-логику.
4. Не добавлять React Router, если его нет в текущей архитектуре.
5. Не добавлять внешний state manager.
6. Не добавлять тяжелый UI-kit без необходимости.
7. Основной frontend остается SPA на React + TypeScript + Vite.
8. Если backend не отдает часть данных, использовать временные mock/calculated values только в UI-слое и явно отделить их от реальных API-данных.
9. После изменений обязательно выполнить:

```bash
cd web
npm install
npm run build
```

Если build невозможен из-за отсутствия Node/npm в окружении — зафиксировать это в отчете.

# 6. Панель сводок пасеки

4. Всего алертов.
5. Средний прирост веса за сутки.

Цветовая логика:

- зеленый — норма;
- amber/orange — предупреждение;
- красный — критичный алерт;
- серый — нет данных.

# 10. Непривязанные устройства

Сделать блок `Непривязанные устройства`.

Показывать устройства, которые backend видит в пасеке, но которые не привязаны к улью.

Карточки/строки:

```text
HM-ESP32-7C3A   HiveNode Mini S   5 мин назад    [Привязать]
HM-ESP32-9F8B   HiveNode Pro      28 мин назад   [Привязать]
HM-ESP32-8E10   HiveNode Pro      1 ч назад      [Привязать]
```

При нажатии `Привязать`:

- выбрать улей;
- выбрать режим старых данных:
  - привязать к улью;
  - оставить диагностическими;
  - удалить;
- для удаления требовать подтверждение.

# 13. Сравнительный график изменения веса всех ульев

Это важный блок.

Название:

```text
Сравнительный график изменения веса ульев на пасеке
```

Требование:

На графике должно быть столько линий, сколько ульев на пасеке.

Если на пасеке 24 улья — график должен показывать 24 линии.

Цель: пользователь должен видеть улей, который выбивается из общей среднестатистической ситуации по искривлению линии.

---

## 13.1 Данные графика

График показывает изменение веса ульев за выбранный период:

```text
1 день
10 дней
30 дней
```

Переключатель периода:

```text
[1 день] [10 дней] [30 дней]
```

Для каждого улья линия должна показывать изменение веса относительно начала периода.

Например:

```text
Улей 12: 0 → +0.4 → +1.2 → +2.4 кг
Улей 08: 0 → -0.2 → -0.8 → -1.4 кг
Улей 03: 0 → +0.01 → +0.03 → +0.05 кг
```

---

## 13.2 Визуальная логика графика

- Все линии по умолчанию тонкие и полупрозрачные.
- Средняя линия по пасеке может быть более темной/пунктирной.
- При hover линия подсвечивается.
- При click линия становится выбранной.
- При выборе линии показывается tooltip/card с информацией об улье.

Tooltip должен показывать минимум:

```text
Улей 12
Изменение: +2.41 кг
Время: 15:00
```

Лучше показывать:

```text
Улей 12
Тип: Дадан с магазином
Изменение за период: +2.41 кг
Текущий вес: 54.8 кг
Температура: 32.1°C
Статус: Активен
```

---

## 13.3 Действие при клике на линию

При клике на линию графика:

1. Выбранный улей становится активным.
2. Строка этого улья в таблице подсвечивается.
3. Справа открывается боковая панель выбранного улья.
4. В боковой панели показываются детали выбранного улья.

---

## 13.4 Если невозможно показать 24 линии красиво

Если библиотека графиков или текущая реализация не позволяет удобно показать много линий:

- всё равно реализовать multi-line chart;
- добавить фильтр/поиск по улью;
- добавить кнопку `Показать только проблемные`;
- добавить выделение выбранного улья;
- остальные линии делать бледными.

Не заменять 24 линии одной суммарной линией.

Суммарная/средняя линия может быть дополнительной, но не вместо линий ульев.

# 14. Боковая панель выбранного улья

Информацию по отдельному выбранному улью не показывать как главный центральный блок dashboard.

Она должна открываться в отдельной правой панели:

- боковая панель;
- drawer;
- slide-over;
- скрывающаяся панель.

Панель открывается:

- при клике на строку улья;
- при клике на линию улья на сравнительном графике;
- при клике на событие/алерт, связанный с ульем.

Панель должна закрываться кнопкой `X`.

---

## 14.1 Содержимое боковой панели

Заголовок:

```text
← Улей 12    Активен        X
```

Вкладки:

```text
Обзор | Графики | История
```

Блок предупреждений:

```text
Есть предупреждения
Улей открыт сегодня в 09:35
Все алерты (1)
```

Картинка улья:

- картинка должна зависеть от типа улья;
- если тип `Дадан` — картинка Дадан;
- если `Дадан с магазином` — Дадан с магазином;
- если `Рута` — Рута;
- если `Лежак` — лежак;
- если `Украинский` — украинский тип;
- если `Колода` — колода;
- если тип неизвестен — универсальная картинка улья.

Поля:

```text
Тип улья: Дадан с магазином
Устройство: HM-ESP32-12A1
Установлен: 15.04.2025
Местоположение: Участок 2
```

Показатели:

```text
Вес: 54.8 кг
Изменение за 24ч: +2.41 кг
Температура: 32.1°C
Влажность: 62%
Батарея: 78% / 3.7V
RSSI: -68 dBm / Отличный
Последнее измерение: Сегодня, 09:36
```

График:

```text
Изменение веса за последние 24 часа
```

Последние события:

```text
Улей открыт                 Сегодня, 09:35
Изменение веса +2.41 кг     Сегодня, 09:00
Температура в норме         Сегодня, 08:00
Низкий заряд устройства     Вчера, 22:15
```

Кнопки внизу:

```text
[Открыть графики] [Перейти к улью]
```


# 20. Графики

Если в проекте уже есть библиотека графиков — использовать ее.

Если библиотеки нет, варианты:

1. Использовать SVG вручную для MVP.
2. Добавить lightweight chart library только если это оправдано.
3. Если добавляется библиотека, выбрать одну и не перегружать проект.

Требование к multi-line chart:

- 24 линии для 24 ульев;
- hover tooltip;
- click selection;
- выделение выбранной линии;
- открытие боковой панели улья при клике;
- остальные линии приглушаются при выборе.

# 21. Поведение выбранного улья

В состоянии frontend хранить:

```ts
selectedHiveId: string | null
isHiveDrawerOpen: boolean
selectedPeriod: '1d' | '10d' | '30d'
```

При клике на строку улья:

```ts
setSelectedHiveId(hive.id)
setHiveDrawerOpen(true)
```

При клике на линию графика:

```ts
setSelectedHiveId(series.hiveId)
setHiveDrawerOpen(true)
```

В таблице выбранный улей подсветить.

На графике выбранную линию сделать ярче/толще.

# 23. Definition of Done

Работа считается выполненной, если:

1. Главный экран выглядит как dashboard пасеки.
2. Есть панель сводок пасеки.
3. Есть блок предупреждений/алертов.
4. Есть блок событий с важными/запланированными/просроченными.
5. Список ульев показывает изменение веса за сутки как главную метрику.
6. Слабый прирост, отрицательный прирост и другие алерты визуально выделены.
7. Есть блок непривязанных устройств.
8. Есть блок текущей погоды и погодных предупреждений.
9. Есть блок советов/подсказок.
10. Общий график показывает отдельные линии по каждому улью, а не одну суммарную линию.
11. При клике на линию графика выбирается соответствующий улей.
12. При выборе улья открывается боковая панель с деталями.
13. Боковая панель показывает предупреждения, телеметрию, график, события и картинку улья.
14. Картинка улья зависит от типа улья, насколько это возможно в текущей реализации.
15. Empty states сделаны понятно.
16. Backend API не изменен.
17. `npm run build` проходит успешно или причина невозможности проверки описана.
