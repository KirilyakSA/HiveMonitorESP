# MVP web frontend

## Реализованный MVP

Реализован первый рабочий интерфейс под текущий backend и firmware provisioning flow.

Функции:

- регистрация;
- вход;
- выход;
- список организаций;
- создание организации;
- список пасек организации;
- создание пасеки;
- список ульев пасеки;
- создание улья;
- список непривязанных устройств пасеки;
- привязка устройства к улью;
- выбор режима старой телеметрии;
- latest telemetry выбранного улья;
- простой график history telemetry за 24 часа;
- журнал событий пасеки;
- журнал событий улья.

## Проверяемый сценарий

1. Запустить backend и mqtt-ingestion-service.
2. Запустить web:

```bash
cd web
npm install
npm run dev
```

3. Зарегистрировать пользователя.
4. Создать организацию.
5. Создать пасеку.
6. Создать улей.
7. Настроить firmware с `apiaryId` пасеки или отправить MQTT telemetry в apiary-aware topic.
8. Устройство появляется в "Непривязанные устройства".
9. Привязать устройство к улью.
10. Открыть улей и увидеть latest/history telemetry.
11. Отправить MQTT event/status и увидеть журнал.

## Что не входит в MVP

- Отправка команд устройству.
- OTA управление.
- Push/Telegram настройки.
- Alert rules.
- Планировщик задач.
- Ручное создание событий.
- Схема расположения ульев.
- Weather analytics.
- Billing/tariffs UI.
- i18n.

## Definition of Done MVP

- UI запускается через Vite.
- Auth работает с backend access token.
- CRUD организации/пасеки/улья работает через backend API.
- Устройства из `GET /apiaries/{id}/devices/unassigned` видны пользователю.
- Привязка устройства вызывает backend endpoint.
- Latest telemetry отображается без подмены отсутствующих данных нулями.
- History telemetry отображается простым графиком.
- Apiary/hive events отображаются из backend journal endpoints.
- UI адаптируется до мобильной ширины.
